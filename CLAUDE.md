# プロジェクトルール

## デザイン

- アプリ作成時は必ずパステル系の色合いを使用する
  - 例: `#FFD1DC`（ピンク）、`#B5EAD7`（ミント）、`#C7CEEA`（ラベンダー）、`#FFDAC1`（ピーチ）、`#E2F0CB`（ライトグリーン）
  - 彩度を抑えた柔らかいトーンを基調とすること
  - 強調色も彩度高めの原色は避け、パステル系の中から選ぶ

## 技術スタック

- ブラウザアプリはHTMLで作成する（HTML / CSS / JavaScript）
  - 単一HTMLファイルでの完結を基本とする
  - 外部フレームワークを使う場合もパステルデザインルールを維持すること

## サービス利用

- 課金が必要なサービスは使用しない
  - 外部API、クラウドサービス、有料ライブラリ等、費用が発生するものは一切使わない
  - 無料枠がある場合でもクレジットカード登録が必要なものは避ける
  - 代替手段として、無料・オープンソースのツールや自己完結した実装を優先する

## ドキュメント

- 新しい機能を追加したときは必ず使い方の説明をする
  - ユーザーへの返答の中で、追加した機能の操作手順を明記すること
  - スクリーンショットが取れない場合は、箇条書きや番号付きリストで手順を説明すること
